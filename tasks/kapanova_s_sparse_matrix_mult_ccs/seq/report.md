## 1. Контекст
Последовательная реализация умножения двух разреженных матриц с элементами типа double, хранящихся в формате Column Compressed Storage служит эталоном корректности и базой для оценки ускорения всех последующих реализаций.

## 2. Постановка задачи

**Входные данные (`InType`):**
- Пара матриц `A` и `B` в формате `CCSMatrix`.
- Структура `CCSMatrix`:
  - `rows`, `cols`: Количество строк и столбцов
  - `nnz`: Количество ненулевых элементов 
  - `col_ptrs`: Массив индексов в `values` и `row_indices`, указывающих на начало данных для каждого столбца. Размер `cols + 1`.
  - `row_indices`: Массив строчных индексов для каждого элемента. Размер `nnz`.
  - `values`: Массив значений элементов. Размер `nnz`.

**Выходные данные (`OutType`):**
- Результирующая матрица `C = A * B` в формате `CCSMatrix`.

**Ограничения и крайние случаи:**

1. **Совместимость размерностей:**
   - `A.cols` должно быть равно `B.rows`. Несовместимые матрицы отбрасываются на этапе валидации.
2. **Ненулевые размерности:**
   - `A.rows > 0`, `A.cols > 0`, `B.rows > 0`, `B.cols > 0`. Матрицы с нулевой размерностью отбрасываются на этапе валидации.
3. **Корректность CCS-структуры:**
   - Для обеих матриц проверяется целостность формата: размер `col_ptrs` равен `cols + 1`, `col_ptrs[0] == 0`, `col_ptrs[cols] == nnz`, размеры `values` и `row_indices` равны `nnz`. Нарушения отбрасываются на этапе валидации.
4. **Полностью нулевые матрицы:**
   - `nnz == 0` допустимо для любой из матриц. Результат — нулевая матрица `C` с корректными размерностями. Проверяется вычислительным тестом.
5. **Прямоугольные матрицы:**
   - `A` и `B` могут быть неквадратными. `C` будет иметь размер `A.rows × B.cols`. Проверяется вычислительным тестом.

Полный перечень проверок и эталонные значения для каждого случая приведены в разделе 5 

## 3. Базовый алгоритм
Пошаговое описание:
1.  **Инициализация:** Для матрицы `C` устанавливаются размеры `A.rows × B.cols`. Выделяется память под `col_ptrs`.
2.  **Внешний цикл (по столбцам `B`):**
    *   Для каждого столбца `j` матрицы `B` создаётся временный вектор`accum` размера `A.rows` и флаги `nz_elem_rows` для фиксации изменённых строк.
    *   **Средний цикл (по элементам столбца `j` матрицы `B`):**
        *   Для каждого ненулевого элемента `B(k, j)` со значением `b_val` и строкой `ind = k`:
        *   **Внутренний цикл (по элементам столбца `k` матрицы `A`):**
            *   Для каждого ненулевого элемента `A(i, k)` со значением `a_val` обновляется вектор `accum[i] += a_val * b_val`. Строка `i` помечается как изменённая.
    *   **Формирование столбца `C`:**
        *   Индексы изменённых строк сортируются по возрастанию.
        *   Все ненулевые значения из `accum` для этих строк записываются в `C.values`, а индексы — в `C.row_indices`.
        *   `accum` и флаги  сбрасываются для следующего столбца.
3.  **Фиксация результата:** в `C.col_ptrs` записываются позиции начала каждого столбца, последний элемент `col_ptrs` равен `C.nnz`.

**Асимптотическая сложность:**
- **Время:** В худшем случае — `O(A.cols * A.rows + A.nnz * (макс. элементов в столбце B))`. Время сильно зависит от заполненности матриц и распределения ненулевых элементов.
- **Память:** `O(A.rows + C.nnz)`, не считая входных данных. Основное потребление — временный вектор `accum`.

**Инварианты и критерий корректности:**
1.  Результат умножения `C` (в пределах точности `eps = 1e-10`) совпадает с эталонным результатом
2.  Выходная матрица - в корректном CCS-формате: индексы в пределах строк, col_ptrs не убывает и col_ptrs[cols] == nnz
3.  Нулевые значения в результат не записываются

## 4. Детали реализации
**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

- **`ValidationImpl`:** Выполняет строгую проверку входных данных на соответствие ограничениям из раздела 2.
- **`PreProcessingImpl`:** Пустая операция, так как данные не требуют предобработки.
- **`PostProcessingImpl`:** Пустая, так как результат RunImpl уже является финальным.
- **`RunImpl`:**
  - Содержит основную логику, описанную в разделе 3.
  - Временный вектор `accum` и вектор флагов `nz_elem_rows` переиспользуются между столбцами, что позволяет избежать повторных выделений памяти.
  - Сортировка `std::ranges::sort(nnz_rows)` гарантирует правильный порядок индексов в столбце `C`.
  - Ключевой фрагмент: 
```cpp
//seq/src/ops_seq.cpp 
for (size_t j = 0; j < b.cols; ++j) {
  c.col_ptrs[j] = c.values.size();
  for (size_t k = b.col_ptrs[j]; k < b.col_ptrs[j + 1]; ++k) {
    size_t ind = b.row_indices[k];
    double b_val = b.values[k];
    for (size_t zc = a.col_ptrs[ind]; zc < a.col_ptrs[ind + 1]; ++zc) {
      size_t i = a.row_indices[zc];
      accum[i] += a.values[zc] * b_val;   // основная вычислительная нагрузка
      if (!nz_elem_rows[i]) {
        nz_elem_rows[i] = true;
        nnz_rows.push_back(i);
      }
    }
  }
  std::ranges::sort(nnz_rows);
  for (size_t i : nnz_rows) {
    if (accum[i] != 0.0) {
      c.row_indices.push_back(i);
      c.values.push_back(accum[i]);
    }
    accum[i] = 0.0;
    nz_elem_rows[i] = false;
  }
  nnz_rows.clear();
}
```
Внешний цикл идёт по столбцам B. Для каждого столбца B средний цикл перебирает его ненулевые элементы, а внутренний цикл — соответствующий столбец A. Результат накапливается во временном векторе accum. Флаги nz_elem_rows отслеживают, какие строки были изменены — это позволяет потом не проверять весь accum на нули, а обработать только изменённые позиции. После сортировки измененных строк ненулевые значения записываются в C, а accum и флаги сбрасываются.

## 5. Проверка корректности
Корректность проверяется с помощью набора фиксированных функциональных тестов, сравнивающих вывод RunImpl с предварительно вычисленным эталонным результатом (expected_result_).

Использованные функциональные тесты:

1. Малые матрицы 2x2 (Test 1): Базовый случай с диагональными матрицами.
2. Матрицы 3x3 со случайной структурой (Test 2 & 3).
3. Прямоугольные матрицы 2x3 и 3x2 (Test 4): Проверка корректности при A.cols != A.rows.
4. (Test 5): Проверка крайнего случая, когда матрица A полностью нулевая. Ожидаемый результат — нулевая матрица.
Все тесты проходят успешно. Сравнение результатов происходит с допуском eps = 1e-10.

**Проверяется в тестах:**
- `rows`, `cols` — точное совпадение размерностей.
- `nnz` — точное совпадение количества ненулевых элементов.
- `col_ptrs` — полное поэлементное совпадение массива указателей.
- `row_indices` — полное поэлементное совпадение строчных индексов.
- `values` — поэлементное сравнение с допуском `1e-10`.
